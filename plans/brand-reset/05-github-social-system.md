# GitHub and Social System

## Goal
Turn shipping activity into audience growth and trust.

## Platform Roles
- GitHub: proof of work and technical trust.
- X: discovery and daily conversation.
- YouTube: deeper walkthrough and search-friendly authority.
- Twitch: live build accountability and clip source.
- Blog: canonical long-form artifact.

## GitHub Profile Structure
- Who you are.
- What you are building now.
- Pinned repos by lane: app, game, CLI.
- CTA: newsletter and follow links.

## Repo Hygiene Standards
- Clear one-line description.
- README with problem, setup, current status, roadmap.
- Release notes or changelog updates for every meaningful ship.

## Weekly Content Loop
- One build artifact per week drives all publishing.
- Transform artifact into:
  - one blog post,
  - one X thread,
  - one short video,
  - one stream talking point.

## X Post Formats (Repeatable)
- Proof post with screenshot and metric.
- Mini thread with decision breakdown.
- Failure postmortem.
- Before/after optimization post.
- Devlog recap.

## YouTube/Twitch Workflow
- One weekly stream (90-120 min).
- Clip 2-4 moments into shorts.
- Publish one recap video with strongest lesson.

## Engagement Protocol
- Daily 30-45 minutes split into two sessions.
- Reply to comments within 24 hours.
- Leave thoughtful replies on relevant builder accounts.
- Weekly outreach to 2-3 potential collaborators.

## CTA System
- Primary: follow the build.
- Secondary: join weekly notes.
- Tertiary: star/watch active repositories.

## Guardrails
- No generic inspiration posting.
- No vanity-only announcements without technical substance.
- No overcommitment to service content.

## Milestones
- 30 days: content consistency and top-format discovery.
- 60 days: repeatable growth loop from releases.
- 90 days: stable weekly system with measurable follower growth.
